1. uwsgi는 서버를 계속 실행시키는 역할을 한다. 도커로 이미 해당 역할을 수행하고 있어서 추가하지 않아도 된다. 장고 코드를 바꾸었을 때 자체적으로 다시 runserver가 실행되는 이유는 장고 자체적으로 이런 cgi 역할을 하는 것이 있어서이다.

2. 데이터베이스에 뭔가 작성하려고 할 때 오류가 안 날 것이 확실해 보여도 try-except 쓰기

3. admin.site.register(모델이름)을 admin.py에 쓰면 따로 커스텀하지 않은 관리자 페이지가 만들어진다.

4. admin 파일의 변경이 적용되는 데에 시간이 좀 걸리거나 서버를 재시작해야 하는 듯하다.

5. 관리자 페이지에서 objects has no attribute 메시지가 뜨는데 해당 모델에 없는 필드라면 매직 메서드를 확인하기. 복붙하는 과정에서 실수로 수정을 덜 했을 수 있다.

6. 인앱결제 과정은 유저가 결제를 하면 영수증이 서버에 전달되고, 서버는 암호화된 영수증을 API 등을 사용해 검증한 후 반영한다. 구글 같은 경우는 시간 내로 검증하지 않으면 (실서버 3일, 테스트서버 5분) 해당 거래가 무효 처리된다. 정기적으로 결제되는 경우 최초 1회만 검증하면 된다.
