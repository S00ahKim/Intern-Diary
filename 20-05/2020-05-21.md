2. 해당 객체를 이해하는 데에 필요한 모든 정보는 객체 안에 있어야 한다.
3. 쉽고, 간단하고, 이해하기 쉽게 짜려고 할 것
- 기본적인 API 개발이 끝나고 나서 전임자분이 짜둔 코드 가운데 이해가 가지 않거나 어떤 부분에서만 쓰이고 있는 부분들을 알아보기 위해 검색을 해봤다. 쿼리 최적화도 시작해야겠다고 어렴풋이 생각하고 있었는데, 그 부분들이 캐싱과 최적화에 대한 부분이었다. related 관련 파라미터들은 잘못 설정되어 있어서 다시 만드는 게 나을 것 같았다. 그러면서 찾다가 장고 독스에서 설계 철학을 찾았는데, 여기서 제일 눈여겨볼만한 것을 정리해보면 위와 같다.
