### nginx
1. nginx 멈추기 
마스터 프로세스를 netstat으로 pid를 확인한 후 바로 kill 해버리면 자식 프로세스들이 자꾸 대체하기 때문에 완벽히 죽일 수 없다. 그래서 nginx stop 같은 커맨드를 사용하는 것임. 임의의 pid를 죽이지 말라고 많은 사람들이 말하고 있다. 하지만 이미 그렇게 하고 있다면 + nginx stop도 듣지 않는다면 프로세스 종료 명령어를 사용한다.
```
pkill nginx
```

2. 실행 중인 pid를 보는 방법
```
ps aux -P
```
- ps 명령어는 현재 시스템에서 돌고있는 프로세스를 보여주는 가장 기본적인 명령어
- htop같이 실시간으로 시스템 자원을 출력하는 명령과는 달리 ps 명령어를 입력한 순간의 정보(user id, cpu 사용량, memory 사용량, 사용한 명령이들 등)를 보여줌.
- ps 명령은 옵션 입력 방법이 bsd 스타일과 unix 스타일, gnu 스타일로 나뉩니다. bsd 스타일은 'ps aux'와 같이 대쉬가 없는 스타일인 반면 unix 스타일은 'ps -ef' 처럼 옵션앞에 대쉬를 넣어 옵션임을 표시하는 스타일입니다. gnu 스타일은 대쉬를 두개 넣는 스타일로 'ps --help' 처럼 사용합니다. 동일하게 문자 u를 사용하는 옵션이라도 대쉬의 유무에 따라 의미가 달라집니다.
